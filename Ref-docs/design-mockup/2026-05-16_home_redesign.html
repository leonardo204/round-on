🌙 다크 모드 (Winter, 시스템 자동 대응)

주요 변경 사항 (검수 포인트)

  • 전체 화면 사용: Color(.systemGroupedBackground).ignoresSafeArea() 시스템 적응형 + .toolbarBackground(.visible) 명시 → 다크 모드에서도 검은 띠 없이 자연스러운 그루핑
  • Hero CTA: 평범한 풀폭 버튼 → 그라데이션 카드 + 한글 카피("오늘의 라운드를 시작해 볼까요?") + 작은 알약 버튼. iOS Health/Fitness 패턴.
  • 이번 달 요약: 작은 메트릭 카드 2장 (라운드 수 · 평균 타수). 통계 화면 진입 전 미리보기.
  • 최근 라운드: 단일 카드 그룹(insetGrouped) — 아이콘 + 골프장명 + 메타 + 총타수 + par 차이(±N). 색상으로 잘함/못함 즉시 인지.
  • 상단 네비게이션: large title + 우측 두 액션 버튼(통계 / 설정) 원형 그레이 캡슐 — Apple 표준 패턴.
  • 아이콘: chart.bar.fill → monitoring (활동 추적 의미 명확), flag (라운드), play_arrow (시작) — SF Symbols 의미 명확.
  • 타이포그래피: large title 34pt/bold, hero 28pt/bold, metric 26pt/bold + tabular-nums, 본문 15pt/regular — SF Pro Display 풀스케일.
  • 색상: Spring 그린 단일 accent, par-diff 5단계 색상(eagle 보라/birdie 초록/par 회색/bogey 주황/doublePlus 빨강) 일관 적용.
  • 다크 모드: 동일 레이아웃 자동 대응 (systemBackground 시스템 토큰). Spring 그린은 다크에서 약간 밝은 변형(#66BB6A) 사용.

현재 (20년 전 같다는 평가)

  • 흰 배경 + 큰 초록 버튼 1개 + 빈 상태
  • 상하단 검은 띠 (다크 모드 대응 미흡)
  • 아이콘만 있는 통계 진입 (의미 불명)

제안

  • Hero 카드 + 메트릭 + 라운드 리스트 = "한눈에 의미 있는 정보"
  • 전체 영역 시스템 색 + 다크 자동
  • 스코어 색상 마커로 시각적 인지 강화
